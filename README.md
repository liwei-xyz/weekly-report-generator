# 职场周报生成器 (Weekly Report Generator)

> 帮你把本周工作整理成一份结构清晰、内容扎实的向上汇报周报

---

## ✨ 特性

- **多行业模板**：支持技术研发、产品经理、运营/市场、销售/商务、设计/创意、通用 6 种行业模板
- **智能 Onboarding**：首次使用自动收集用户信息，生成个性化周报
- **上级点评**：AI 根据行业专业知识，像上级一样给出针对性建议
- **反馈收集**：每次生成后主动收集反馈，持续优化周报质量
- **多格式输出**：纯文本（兼容邮件/飞书/微信）、Markdown（适合 Obsidian/Notion）、飞书文档
- **默认配置**：汇报对象默认直属上级，输出格式默认纯文本，可随时修改

## 🚀 安装后第一步

**安装完成后，请立即对我说：「写周报」**

我会引导你完成 3 个简单问题（1 分钟搞定）：
1. 所在行业
2. 工作职责
3. 关注重点

配置完成后，以后每次说「写周报」就能自动生成！

---

## 📋 使用流程

1. **首次使用**：说「写周报」→ 完成 Onboarding 配置
2. **以后每次**：说「写周报」→ 自动生成周报
3. **修改配置**：随时说「改成 Markdown 格式」等

## 📝 周报格式示例

### 纯文本格式（默认）

```
周报 2026-03-29

一、重点工作（1-3 项）

1、{一句话，融合背景 + 方案 + 现状，不超过 30 字}
1）{分点 1：补充细节，不超过 30 字}
2）{分点 2：补充细节，不超过 30 字}

2、{一句话，融合背景 + 方案 + 现状，不超过 30 字}
1）{分点 1}
2）{分点 2}

二、反思总结

1、对公司/团队的建议
1）{一句话，不超过 30 字}

2、个人可以改进的
1）{一句话，不超过 30 字}

三、下周计划

1、未来的计划与尝试方向
1）{一句话，不超过 30 字}

2、需要哪些支持
1）{一句话，不超过 30 字，无则省略此项}
```

### Markdown 格式

```markdown
# 周报 2026-03-29

## 一、重点工作

### 1. {工作标题}
- **现状**：{完成/进行中/阻塞}
- **结论**：{一句话结论}
- **数据**：{具体数据支撑}
- **卡点/风险**：{如有}
- **需要支持**：{如有}
```

## ⚙️ 配置说明

### 默认配置

| 配置项 | 默认值 | 修改方式 |
|--------|--------|----------|
| 汇报对象 | 直属上级 | 说"改成老板"或"改成团队全员" |
| 输出格式 | 纯文本 | 说"改成 Markdown"或"改成飞书文档" |

### 配置文件位置

`~/.weekly-report/config.json`

```json
{
  "onboardingComplete": true,
  "industry": "技术研发",
  "role": "游戏服务器开发工程师",
  "reportTo": "直属上级",
  "focusAreas": ["工作成果/数据", "风险与卡点"],
  "outputFormat": "纯文本",
  "createdAt": "2026-03-29T12:00:00.000Z"
}
```

## 🎓 示例

### 示例 1：技术研发

```
周报 2026-03-29

一、重点工作

1、游戏服务器性能优化完成，QPS 从 1000 提升到 3000
1）优化数据库连接池配置，减少等待时间
2）重构核心战斗逻辑，降低 CPU 占用

2、新副本系统开发完成 80%，预计下周上线
1）完成副本匹配逻辑和奖励发放
2）卡点：测试环境资源不足，需协调

二、反思总结

1、对公司/团队的建议
1）建议建立性能测试基准，定期回归

2、个人可以改进的
1）任务估算偏乐观，需预留缓冲时间

三、下周计划

1、未来的计划与尝试方向
1）完成新副本系统上线，学习 AI 编程课程

2、需要哪些支持
1）协调测试服务器资源用于性能压测

四、上级点评

【成绩肯定】性能优化成果显著，QPS 提升 3 倍，说明技术选型和实施方案都很到位。

【改进建议】新副本系统开发进度 80%，要警惕最后 20% 的延期风险，建议提前和测试团队沟通排期。

【下一步方向】性能优化方案建议写成技术文档，团队内分享，避免其他人重复踩坑。
```

## 🤝 反馈收集

每次生成周报后，AI 会主动收集反馈：
- 长度是否合适
- 重点是否突出
- 格式是否符合预期
- 有无遗漏工作

反馈会保存到 `~/.weekly-report/feedback.json`，用于优化下一次生成。
